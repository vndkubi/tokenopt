## Key Findings

### compiled-packet (TokenOpt only) — winner on all metrics
- Quality 0.990 average (11/12 perfect, 1/12 at 0.875)
- Token range: 1 261 – 3 302 tok (narrow repo) · 1 454 – 2 874 tok (large repo)
- 1 MCP call replaces 3–6 raw tool calls
- vs baseline: −64 % tokens on average, +15 % quality

### tokenopt-codegraph — same quality, higher cost
- Quality identical to compiled-packet (0.990)
- Tokens 2.4–3.7× higher because CodeGraph returns full file snippets:
  - doughnut/implement: 60 669 chars tool output from CodeGraph
  - doughnut/write-unittest: 51 454 chars
- The rubric tests **inventory-level** signals (build tool, source dirs, file list). TokenOpt already fills all slots. CodeGraph adds symbol/call-edge data the rubric doesn't score.
- **When joint pays off**: rubric testing symbol-level accuracy (callers of changed method, exact test class, call chain depth) — data only CodeGraph index has.

### codegraph-only — weakest overall
- Quality 0.709 — fails on build-handoff (0.500) and review-diff (0.375)
- build-handoff: CodeGraph doesn't expose build-tool metadata → rubric fails `build_tool` and `build_facts` checks
- review-diff: CodeGraph response doesn't surface `source_dirs` in a form the signal parser catches → `affected_modules` and `key_files` rubric checks fail
- For small repos (tokenopt 113 files): tokens competitive with baseline (3 207 vs 6 312), but quality much lower
- For large repos (doughnut 1 835 files): tokens similar to baseline on broad tasks, explodes on coding tasks (13k tok for implement)
- **Use case**: symbol/flow navigation when you already know the file/class — not broad repo entry point

### baseline — decent for broad tasks, doesn't scale
- Reads 23 273 chars raw `rg --files` listing every run
- Quality 1.000 for broad tasks (build-handoff, investigate, research-business) because those only need file listing + root file reads
- Falls to 0.857 / 0.750 on coding/review tasks (no structured evidence of source dirs or important files)
- Would degrade significantly at 10k–70k file repos (listing truncation, timeout risk)

## Reproducing

```bash
# Run all 4 modes in parallel
node dist/cli.js benchmark daily --repo /path/to/repo --task all --mode baseline &
node dist/cli.js benchmark daily --repo /path/to/repo --task all --mode codegraph-only &
node dist/cli.js benchmark daily --repo /path/to/repo --task all --mode compiled-packet &
node dist/cli.js benchmark daily --repo /path/to/repo --task all --mode tokenopt-codegraph &
wait
```

Set `CODEGRAPH_CLI_PATH` env var if code-graph is not at `../code-graph/dist/cli.js` relative to tokenopt root.
