# Security Policy

## Supported Versions

TokenOpt is pre-1.0 software. Security fixes are handled for the current `main`
branch and the latest published `0.x` release when one exists.

## Reporting A Vulnerability

Do not include exploit details, private logs, repository contents, API keys, or
other secrets in a public issue.

Preferred path:

1. Use GitHub private vulnerability reporting for this repository if it is
   enabled.
2. If private reporting is not enabled, open a minimal public issue asking for a
   security contact. Include only the affected component and a high-level impact
   summary.

Useful report details:

- Affected command, MCP tool, hook, or setup surface.
- Host surface, such as Codex, Copilot CLI, or VS Code Copilot Agent.
- Operating system and Node.js version.
- Minimal reproduction steps with redacted paths and secrets.
- Expected impact, such as secret exposure, unsafe command execution, policy
  bypass, or misleading benchmark output.

## Scope

In scope:

- TokenOpt CLI commands.
- TokenOpt MCP server behavior.
- Codex and Copilot setup output generated by TokenOpt.
- Hook policy decisions and command output compression.
- Benchmark runners when they could expose sensitive data or execute unintended
  commands.

Out of scope:

- Vulnerabilities in third-party coding agents, MCP clients, or repository
  hosting services.
- Findings that require publishing private target repository contents.
- Reports based only on missing hard enforcement in a host that does not support
  hard enforcement.

## Handling Expectations

This project is maintained on a best-effort basis. Confirmed issues should be
fixed with the smallest compatible change, a regression test where practical,
and a note in release or PR documentation describing the affected surface.
